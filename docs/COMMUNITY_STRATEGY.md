### 3.1 Governance Transition

#### Community DAO
```markdown
STELLAR-PROVENANCE DAO

STRUCTURE:
- Governance token: STELLAR
- Voting: 1 token = 1 vote
- Proposal threshold: 1000 tokens
- Vote duration: 3 days
- Execution: 2-day timelock

RESPONSIBILITIES:
- Approve new features
- Manage treasury (~10% of fees)
- Elect core team
- Set marketplace rules
- Approve partnerships
```

#### Voting Process
```markdown
STAGE 1: TEMPERATURE CHECK (Off-chain, 1 day)
- Community gauges interest
- Feedback collected

STAGE 2: PROPOSAL (On-chain, 3 days)
- Formal proposal submitted
- Community votes
- >50% required to pass

STAGE 3: EXECUTION (2-day timelock)
- Approved proposals queued
- Team executes after timelock
- Results announced
```

#### Governance Incentives
```markdown
VOTER REWARDS
- Vote in 3+ proposals per month â†’ 10 tokens
- Propose feature â†’ 50 tokens (if accepted)
- Active in governance discussions â†’ 5 tokens/week

PARTICIPATION TIERS:
- Passive: Watch & vote
- Active: Vote + comment
- Engaged: Vote + comment + propose
- Leadership: Lead working groups
```
